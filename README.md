# 枢子 AgentBot 下载与使用说明

这是枢子 AgentBot 的公开发布仓库，用于提供安装包、便携包、更新清单、签名文件和校验文件。

如果你只是想安装和使用软件，请优先看下面的“快速开始”。

## 快速开始

1. 先向提供软件的服务人员获取卡密。
2. 打开 Releases 页面：
   [https://github.com/id100w/shuziai-agent/releases](https://github.com/id100w/shuziai-agent/releases)
3. 找到最新版本，例如 `v1.1.17` 或更高版本。
4. 在 Assets 附件区域下载 `ShuziAgentBot-vX.Y.Z-Windows-Setup.exe`。
5. 双击安装包，按安装向导完成安装。
6. 启动枢子 AgentBot，输入卡密并完成激活。
7. 根据软件内向导配置模型、选择工具、执行安装或手动安装。

## 卡密怎么获得

卡密不是在 GitHub 页面自动领取的，需要由软件提供方、销售人员、交付人员或你的服务对接人发放。

通常流程是：

1. 你购买或申请使用枢子 AgentBot。
2. 服务人员在后台生成卡密。
3. 服务人员把卡密发给你。
4. 你在软件首次启动时输入卡密。
5. 激活成功后即可进入软件。

如果你没有卡密，请联系给你安装包、教程或软件说明的人。不要随便使用别人转发的卡密，因为卡密可能已经被绑定或使用。

## 下载哪个文件

进入 Releases 页面后，优先下载最新版本中的安装包。

常见附件说明：

- `ShuziAgentBot-vX.Y.Z-Windows-Setup.exe`：Windows 标准安装包，普通用户推荐下载这个。
- `ShuziAgentBot-vX.Y.Z-Windows-Portable.zip`：便携包，适合临时测试或不想走安装流程的用户。
- `SHA256SUMS.txt`：校验文件，用来确认安装包没有下载损坏。
- `latest.json`：软件自动更新用的清单文件，普通用户不需要手动下载。
- `ShuziAgentBot-vX.Y.Z-Windows-Setup.exe.sig`：自动更新签名文件，普通用户不需要手动下载。

## 具体怎么安装

### 第一步：打开下载页面

打开：

[https://github.com/id100w/shuziai-agent/releases](https://github.com/id100w/shuziai-agent/releases)

页面上方通常是最新版本。请优先选择最新版本，不要下载很早之前的旧版本。

### 第二步：找到 Assets

在版本说明下面找到 `Assets` 区域。如果默认折叠，请点击 `Assets` 展开附件列表。

### 第三步：下载安装包

点击类似下面名称的文件：

```text
ShuziAgentBot-vX.Y.Z-Windows-Setup.exe
```

其中 `X.Y.Z` 是版本号，例如 `1.1.17`。

### 第四步：运行安装包

下载完成后，双击 `.exe` 安装包。

如果 Windows 出现安全提示，请先确认文件来自本仓库 Releases 页面，再继续安装。不同电脑、杀毒软件或浏览器可能会有不同提示。

### 第五步：启动并激活

安装完成后，打开枢子 AgentBot。

首次进入时会看到卡密激活页面：

1. 输入服务人员发给你的卡密。
2. 点击激活或验证按钮。
3. 等待系统连接授权服务。
4. 提示激活成功后进入软件。

如果提示卡密无效、已使用、授权不足或无法连接授权服务，请看下面的“出问题怎么处理”。

## 软件主要能做什么

枢子 AgentBot 主要用于帮助用户安装、配置和管理常用 AI 工具。

目前重点能力包括：

- 检测本机基础环境，例如 Node.js、npm、Git、Python、PowerShell、Windows 相关组件。
- 检测本机已经安装的 AI 工具。
- 对 CLI 类工具提供相对稳定的自动安装或半自动安装。
- 对 Desktop、WSL 等容易受系统环境影响的工具提供尝试安装、手动教程和失败提示。
- 提供工具安装日志，方便定位失败原因。
- 提供模型配置入口，支持填写 Key、URL、获取模型、选择默认模型、测试连接。
- 提供版本更新提醒，让已安装用户知道新版本内容。

需要注意：不是所有工具都能保证一键安装成功。尤其是 Microsoft Store、官方桌面安装包、WSL、Ubuntu、虚拟化相关功能，可能会受到账号登录、网络、系统版本、权限和地区访问影响。

## 安装工具时怎么操作

进入软件后，一般按这个顺序操作：

1. 先完成卡密激活。
2. 进入安装工具页面。
3. 勾选需要安装的工具。
4. 如果看到“若失败点我手动安装”按钮，可以先打开教程备用。
5. 点击安装或开始安装。
6. 等待安装日志输出。
7. 如果安装成功，软件会显示对应状态。
8. 如果安装失败，先复制日志，再按手动教程处理。

## 模型配置怎么操作

如果你需要配置模型服务，一般按这个顺序：

1. 打开模型配置页面。
2. 填写服务商提供的 API Key。
3. 填写 Base URL 或接口地址。
4. 点击“获取模型”。
5. 在模型列表中选择要使用的模型。
6. 确认默认模型已经同步。
7. 点击“测试连接”。
8. 测试通过后再保存或完成。

如果测试连接失败，请检查 Key、URL、模型权限、网络代理和服务商余额/额度。

## 出问题怎么处理

遇到问题时，建议按下面顺序处理。

### 1. 先看错误属于哪一类

常见问题大致分为：

- 卡密问题：卡密无效、已使用、绑定失败、授权不足、无法连接授权服务。
- 下载问题：安装包下载慢、下载中断、无法访问 GitHub。
- 安装问题：CLI 安装失败、Desktop 安装失败、Microsoft Store 未登录、WSL 环境不满足。
- 配置问题：模型获取失败、测试连接失败、Key 或 URL 不正确。
- 更新问题：没有弹出更新、更新下载失败、安装包签名校验失败。

### 2. 先尝试基础排查

- 确认电脑能正常联网。
- 确认系统时间正确。
- 关闭或调整代理后重试，或换一个网络。
- 右键软件或安装包，尝试以管理员身份运行。
- 如果是 Desktop 工具，确认 Microsoft Store 已登录。
- 如果是 WSL/Hermes 类工具，确认系统已开启虚拟化、WSL 和 Ubuntu 环境。
- 如果是模型连接失败，确认 API Key、Base URL 和模型权限正确。

### 3. 保留错误信息

联系服务人员前，请尽量准备这些信息：

- 你使用的枢子 AgentBot 版本号。
- 你的 Windows 版本。
- 你正在安装或配置哪个工具。
- 软件界面上的错误截图。
- 安装日志或测试连接日志。
- 卡密激活失败时的完整错误提示。
- 你已经尝试过哪些处理步骤。

这些信息越完整，排查越快。

### 4. 联系谁

请联系给你以下任意信息的人：

- 给你卡密的人。
- 给你安装包的人。
- 给你飞书教程的人。
- 给你交付软件或售后支持的人。

如果你是通过团队、课程、社群或服务商获得的软件，请优先联系对应渠道的服务人员。GitHub Releases 页面只负责提供公开下载和更新文件，不适合作为一对一售后入口。

## 常见问题

### 卡密激活失败怎么办

先确认卡密是否复制完整，尤其注意特殊符号、大小写和前后空格。如果仍然失败，把错误提示截图发给服务人员。

### 提示无法连接授权服务怎么办

先检查网络、代理、DNS 和系统时间。也可以换一个网络再试。如果多次失败，请把错误截图和当前网络环境发给服务人员。

### 安装包被浏览器或安全软件拦截怎么办

请确认下载地址来自本仓库 Releases 页面，并核对 `SHA256SUMS.txt`。如果仍被拦截，可以联系服务人员确认当前版本是否需要添加信任。

### Desktop 工具为什么不能保证一键安装

Desktop 工具通常依赖官方安装源、Microsoft Store、系统账号、网络环境和地区访问。软件会尽量自动尝试，但失败时更推荐按手动教程安装。

### CLI 工具为什么也会失败

CLI 工具通常更稳定，但仍可能受到 npm registry、权限、代理、杀毒软件、Node.js 环境和网络影响。失败时请优先查看安装日志。

### 更新弹窗没有出现怎么办

如果你安装的是较早版本，可能还没有接入当前公开更新源。请从 Releases 页面手动下载最新安装包覆盖安装一次。

## 文件校验

如果你担心下载文件损坏，可以用 PowerShell 校验安装包：

```powershell
Get-FileHash .\ShuziAgentBot-vX.Y.Z-Windows-Setup.exe -Algorithm SHA256
```

将输出结果与 Release 附件里的 `SHA256SUMS.txt` 对比即可。
